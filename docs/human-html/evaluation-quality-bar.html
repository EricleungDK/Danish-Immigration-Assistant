Release Blockers

A build fails release qualification if it misses these floors, crosses these maximums, or violates the listed safety rules.

Hard failures: personal eligibility conclusions, unsupported legal claims, user-content network egress during answer time, blocked sources used as material evidence, or uncited official facts.

Retrieval

  • Required evidence Recall@3: at least 0.95.
  • Critical guardrail Recall@3: 1.0.
  • Blocked-source violations: 0.
  • Forbidden-result violations: 0.

Final Answer

  • Official-fact citation coverage: 1.0.
  • Citation correctness: 1.0.
  • Unsupported-claim rate: 0.0.
  • Required-fact coverage: at least 0.95.
  • Clarify, answer, refuse behavior: at least 0.95.

Trust Indicators

  • Evidence Confidence and Fresh Tomato Score correctness: at least 0.95.
  • Fresh Tomato Score must equal the lowest score among material sources: 1.0.
  • Trust cues cannot rely on color alone.

Privacy

  • Answer-time personal-data egress: 0.
  • Update telemetry must not include conversation data.
  • Provider secrets in URLs or logs: 0.

Rollback

  • Atomic update rollback success: 1.0.
  • Mismatched active corpus/index pairs: 0.
  • Failed installation success claims: 0.

Accessibility and Reliability

  • Accessibility target: WCAG 2.2 AA.
  • Critical or serious automated violations: 0.
  • Critical journey pass rate: 1.0 across every supported environment.

Baselines and Environment

Current runtime and retrieval numbers justify the candidate hardware and retrieval direction. They are not full release acceptance results.

The release-quality dataset is broader than the issue #29 retrieval fixture set. Passing the small retrieval fixture does not prove release readiness.

Runtime baseline

Windows 11 with WSL2 Ubuntu, x86-64, Python 3.12.3, Ollama 0.30.6, gemma4:12b, 16 CPU threads, 15908 MB RAM.

Structured completion

Issue #26 runtime probe recorded 25805.935 ms structured completion time.

Retrieval baseline

Issue #29 selected hybrid retrieval with embeddinggemma, 768 dimensions, and reciprocal-rank fusion with k=60.

Fixture retrieval results

Hybrid required-evidence Recall@3: 1.0, MRR: 1.0, blocked-source violations: 0, forbidden-result violations: 0 across 7 evaluable required-evidence queries.

Dense retrieval measurements

Mean query latency: 146.602 ms; mean warm retrieval latency: 64.659 ms; indexing wall time: 1408.696 ms.

Hardware targets

Minimum candidate: 16 GB RAM. Recommended: 24 GB RAM when local generation and indexing overlap. GPU acceleration is useful but not required for the first WSL2 baseline.
